# Mojo 🔥 Programming

A living collection of problems in Mojo — started as a way to learn 
the language from scratch, continuing as a deliberate practice to keep 
algorithmic thinking sharp.

The problems here range from first steps to more involved exercises. 
Some were written while figuring out Mojo's ownership model and SIMD 
semantics for the first time. Others are here simply because staying 
fluent in fundamentals is how you stay sharp when you're also building 
something as deep as [Tenmo](https://github.com/ratulb/tenmo).

It is a gym log - algorithmic muscle stretching.
